## üìÅ **File Structure & Components**

### **1. Apollo Client Configuration** (`/src/lib/apollo/client.ts`)
**What it does:** The brain of the authentication system
```typescript
// üîó AUTH LINK - Automatically adds JWT tokens to every GraphQL request
const authLink = setContext(() => {
  const token = localStorage.getItem('excel-pilot-token')
  return {
    headers: { authorization: token ? `Bearer ${token}` : '' }
  }
})

// üõ°Ô∏è ERROR LINK - Handles auth failures and redirects to login
const errorLink = onError(({ graphQLErrors, networkError }) => {
  if (graphQLErrors) {
    for (let err of graphQLErrors) {
      if (err.extensions?.code === 'UNAUTHENTICATED') {
        localStorage.removeItem('excel-pilot-token')
        window.location.href = '/login'  // Redirect to login
      }
    }
  }
})

// HTTP LINK - Connects to GraphQL backend
const httpLink = createHttpLink({
  uri: process.env.NEXT_PUBLIC_GRAPHQL_ENDPOINT || 'http://localhost:4000/graphql',
})
```

**Why this matters:** Every GraphQL request automatically includes the user's JWT token, and if the token expires or becomes invalid, the user is automatically redirected to login.

**Backend Integration:** These match my backend's GraphQL schema perfectly:
- Uses `input` parameter (not separate email/password)
- Returns `accessToken` (not `token`)
- Matches my `AuthPayload` type

```typescript
export const useLogin = () => {
  const login = async (input: LoginInput) => {
    const { data } = await loginMutation({ variables: { input } })
    if (data?.login.accessToken) {
      setToken(data.login.accessToken)  // Save to localStorage

    }
  }
  return { login, loading, error }
}

// üë§ AUTH STATUS HOOK
export const useAuthStatus = () => {
  const { user } = useCurrentUser()
  const token = getToken()
  return {
    isAuthenticated: !!token && !!user,
    user,
    token
  }
}
```

## üîÑ **Authentication Flow - Step by Step**

### **Login Process:**
1. **User enters credentials** ‚Üí Login form
2. **Form submits** ‚Üí `useLogin()` hook
3. **GraphQL mutation** ‚Üí `LOGIN_MUTATION` with `{ input: { email, password } }`
4. **Backend responds** ‚Üí `{ user: {...}, accessToken: "jwt..." }`
5. **Token saved** ‚Üí `localStorage.setItem('excel-pilot-token', token)`
6. **User redirected** ‚Üí Dashboard or intended page
7. **Navigation updates** ‚Üí Shows user avatar and info

### **Authenticated Requests:**
1. **User makes request** ‚Üí Any GraphQL operation
2. **Auth link activates** ‚Üí Automatically adds `Authorization: Bearer <token>`
3. **Backend validates** ‚Üí JWT token verification
4. **Response returned** ‚Üí With user-specific data

### **Token Expiration/Logout:**
1. **Token expires** ‚Üí Backend returns `UNAUTHENTICATED` error
2. **Error link catches** ‚Üí Automatically removes token from localStorage
3. **User redirected** ‚Üí Login page
4. **Navigation updates** ‚Üí Shows "Sign In" button

## üõ†Ô∏è **Key Technical Decisions**

### **Why Apollo Client Links?**
- **Auth Link**: Automatically adds JWT tokens to every request
- **Error Link**: Handles auth failures gracefully
- **Retry Link**: Retries failed requests automatically
- **HTTP Link**: Connects to your GraphQL backend

### **Why localStorage for Tokens?**
- ‚úÖ Persists across browser sessions
- ‚úÖ Automatically included in all requests
- ‚úÖ Easy to clear on logout/expiration
- ‚ö†Ô∏è Note: In production, consider httpOnly cookies for better security

### **Why Custom Hooks?**
- ‚úÖ Reusable across components
- ‚úÖ Encapsulates complex logic
- ‚úÖ Provides consistent error handling
- ‚úÖ Makes components clean and focused

## üîß **Troubleshooting Guide**

### **"Could not find client in context"**
- ‚úÖ **Fixed**: Navigation moved inside `<Providers>` wrapper
- **Cause**: Apollo hooks need to be inside ApolloProvider

### **"NextRouter was not mounted"**
- ‚úÖ **Fixed**: Changed from `next/router` to `next/navigation`
- **Cause**: Using wrong router for Next.js 13+ App Router

### **GraphQL Schema Errors**
- ‚úÖ **Fixed**: Updated mutations to use `input` parameter and `accessToken`
- **Cause**: Frontend schema didn't match backend expectations
